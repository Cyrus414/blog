Python 参数传递方式

位置传递

顾名思义，就是通过对应位置传递参数，传入参数需要考虑传入顺序

>>> def fn(p1, p2, p3):
...     print(p1, p2, p3)
... 
>>> fn(1, 2, 3)
1 2 3
>>> fn(3, 2, 1)
3 2 1

关键字传递

关键字传递可以通过参数名来指定传入参数，不用像位置传递一样严格按参数定义顺序传入

>>> def fn(p1, p2, p3):
...     print(p1, p2, p3)
... 
>>> fn(p1=1, p2=2, p3=3)
1 2 3
>>> fn(p3=3, p2=2, p1=1)
1 2 3

关键字传递可以与位置传递混用，需要注意的是位置传递需要在关键字传递之前使用

>>> def fn(p1, p2, p3):
...     print(p1, p2, p3)
... 
>>> fn(1, 2, p3=3)
1 2 3
>>> fn(1, p3=3, p2=2)
1 2 3

参数默认值(缺省值)

在定义函数时可以将变动很少或者最常用的参数定义默认值，这样调用函数时不传递对应参数即使用默认值。需要注意的时参数默认值一定要放在参数列表的末尾。

>>> def fn(p1, p2, p3=3):
...     print(p1, p2, p3)
... 
>>> fn(1, 2)
1 2 3
>>> fn(1, 2, 4)
1 2 4

当定义的函数存在多个参数默认值时尽量使用关键字传递以保证参宿传递的正确性

>>> def fn(p1, p2=2, p3=3):
...     print(p1, p2, p3)
... 
>>> fn(1, 2, 3)
1 2 3
>>> fn(1, 2)
1 2 3
>>> fn(1, p3=2)
1 2 2

包裹传递

很多时候我们可能不太确定最终传入的参数个数，这时可以使用包裹传递(也可以理解为可变参数)来帮助我们完成任务。

  • 定义函数时在形参前面加上*代表包裹位置参数
>>> def fn(*p):
...     print(p)
... 
>>> fn(1, 2, 3)
(1, 2, 3)

可以看到传入的参数与被自动封装成了一个元组。当然我们也可以和位置混用。

>>> def fn(p1, *p):
...     print(p1, p)
... 
>>> fn(1, 2)
1 (2,)
>>> fn(1, 2, 3, 4)
1 (2, 3, 4)

个人建议如果和位置传递混用的话将位置传递的参数放在前面，当然放在后面也没有问题，但是调用的时候就一定要使用关键字传递了。

>>> def fn(*p, p1):
...     print(p, p1)
... 
>>> fn(1, 2, 3, 4)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: fn() missing 1 required keyword-only argument: 'p1'
>>> fn(1, 2, 3, p1=4)
(1, 2, 3) 4

这样做看着是不是有点蠢， 所以还是把包裹放到后面去吧。

  • 定义函数时在形参前面加上**代表包裹关键字参数
>>> def fn(**p):
...     print(p)
... 
>>> fn(a=1, b=2, c=3)
{'a': 1, 'b': 2, 'c': 3}

可以看到这时传入的关键字参数被自动封装成了一个字典

解包裹

说到包裹传递呢就不得不说一下解包裹，简单的我们可以将解包裹理解为是包裹的一个逆向操作。

  • 在元组或列表前面加上*代表解包裹
>>> def fn1(p1, p2, p3):
...     print(p1, p2, p3)
... 
>>> fn1(1, 2, 3)
1 2 3
>>> def fn2(*p):
...     print(p)
... 
>>> fn2(1, 2, 3)
(1, 2, 3)
>>> def fn3(*p):
...     print(*p)
... 
>>> fn3(1, 2, 3)
1 2 3

fn1和fn2我们已经在前面的位置传递和包裹位置传递中演示过效果了，在调用fn2时会收集所以传入的参数封装为一个元组，我们看到fn3的函数定义和fn2是一样的，但是fn3的运行结果却是和fn1一样，导致这样的结果关键点在*p上，在列表或元组前加上*会将列表或元组中的各个元素展开。通过fn2我们可以知道参数p已经自动被封装成了一个元组，而我们在打印p的时候在前面加上了*进行解包裹，从而达到了与fn1一样的效果。我们再来举个例子理解一下解包裹。

>>> a=[1, 2, 3, 4]
>>> print(*a)
1 2 3 4
>>> b=(1, 2, 3, 4)
>>> print(*b)
1 2 3 4
  • 在字典前面加上**代表解包裹
>>> def fn(**p):
...     print(p)
... 
>>> d = {'a': 1, 'b': 2, 'c': 3}
>>> fn(**d)
{'a': 1, 'b': 2, 'c': 3}

来解释一下，d是我们定义的一个字典，当把d传入fn的时候我们在前面加了**做了一次解包裹，这样就等效的执行了fn(a=1, b=2, c=3)，从而得到了我们介绍关键字包裹传递时用的例子一样的效果。

  • 一个解包裹的小应用
>>> x = {'a': 1, 'b': 2, 'c': 3}
>>> y = {'d': 4, 'e': 5, 'f': 6}
>>> z = {**x, **y}
>>> print(z)
{'a': 1, 'b': 2, 'c': 3, 'd': 4, 'e': 5, 'f': 6}

通过解包裹操作合并了两个字典，是不是很方便呢。

写在最后

我们在定义函数的时候可以将上述讲到的几种传递方式混合使用，但是最好遵循一个基本原则一面带来不必要的麻烦。规则是按照 “位置，关键字，包裹位置，包裹关键字”的顺序。
